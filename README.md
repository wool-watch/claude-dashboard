# Claude Code 使用状況ダッシュボード

Claude Code・Codex CLI・Gemini CLI のローカルセッションログを集計し、
**トークン使用量・推定コスト・操作時間の可視化**と **AIによるセッション振り返り** ができる
ローカル専用 Web ダッシュボードです。

- **DB なし・認証なし・外部送信なし** — ログファイルを直接読んで集計します
- **マルチ CLI 対応** — Claude Code に加えて Codex CLI / Gemini CLI のセッションも取り込み可能
- **自動アーカイブ** — CLI 側でログが消えても閲覧を継続できるようミラーを保持
- **AI 振り返り** — セッションの使い方をハーネス実践の観点で採点し、改善アクションを提案

## セットアップ

```bash
npm install
npm run dev        # http://localhost:3947
```

- ポートを変えたい場合: `npx next dev -p <port>`
- サーバーはデフォルトで `127.0.0.1` のみにバインドします（認証がないため）。
  LAN 内の他端末から使う場合は `-H` オプションの変更に加え、
  [セキュリティ](#セキュリティ)の許可リスト追加が必要です。

起動するだけで `~/.claude/projects` などのログを自動で読み込みます。事前設定は不要です。

## 対応ソース

| ソース | 読み込み元 | 取り込み |
|---|---|---|
| Claude Code | `~/.claude/projects/**/*.jsonl` | 常に有効 |
| Codex CLI | `~/.codex/sessions`（＋ `~/.codex/archived_sessions`）のロールアウト | 設定モーダルで切替 |
| Gemini CLI | `~/.gemini/tmp/<hash>/chats` のチャット記録 | 設定モーダルで切替 |

取り込んだセッションはすべて一覧・詳細・集計・AI振り返り・アーカイブの対象になり、
一覧にはソースバッジ（Claude Code / Codex CLI / Gemini CLI）が表示されます。

> **Gemini CLI 利用者への注意**: Gemini CLI のチャット記録は CLI 側の既定で
> 30日後に自動削除されます。取り込みを有効にしておくと、本ダッシュボードの
> アーカイブが実質的なバックアップになります。

## 画面と使い方

### ダッシュボード（`/`）

利用状況の全体像を表示します。

- **サマリーカード**: 総コスト・トークン・セッション/ターン数・操作時間（今日/今週/今月の内訳付き）
- **時系列チャート**: コスト・トークンの推移（時/日/週/月で粒度切替）
- **カレンダーヒートマップ**: 過去26週の日別アクティビティ
- **曜日×時間帯ヒートマップ**: 利用が多い時間帯の傾向
- **モデル別・ツール別統計**: モデルごとのコスト比率、ツール呼び出し回数
- **振り返りサマリー（AI分析の傾向）**: 分析済みセッションを横断した
  ハーネス実践スコアの週次トレンド・定量メトリクスの推移と、「優先課題の分析」の実行

### プロジェクト（`/projects` → `/projects/[id]`）

- 一覧: プロジェクトごとのセッション数・ターン数・トークン・コスト・操作時間・最終利用。
  見出しの検索アイコンから名前・パスで絞り込み、見出しクリックで並べ替え
- 詳細: そのプロジェクトに絞ったダッシュボード（上記と同じチャート一式）＋セッション一覧

### セッション一覧（`/sessions`）

- 見出しに **絞り込みとソートを内蔵**: タイトル検索・ソース・分析ステータスは
  見出しのフィルタアイコンから、並べ替えは見出しクリックで
- **一括分析**: チェックボックスでセッションを選択し「選択したセッションを分析」で
  AI振り返りをまとめてキュー投入（実行中・待機中のものは選択不可）
- 各行に分析ステータスバッジ（未分析 / 待機中 / 分析中 / 分析済み / 再分析推奨）を表示

### セッション詳細（`/sessions/[id]`）

- コスト・トークン・ターン数・所要時間・操作時間のカード
- **AI振り返り**: 「このセッションを分析する」で実行。結果（スコア・改善アクション・
  定量メトリクス）はローカルに保存され、次回以降は保存済み結果を表示
- **ターン別タイムライン**: プロンプト・使用モデル・ツール呼び出し・トークン内訳・
  コスト・所要時間をターンごとに表示

### ヘッダー共通 UI

- **分析状況**: 分析キューの実行中・待機中を一覧表示。中止・再開もここから。
  前回終了時に未完了の分析が残っていた場合は再開確認が表示されます
- **テーマ切替**: ライト / ダーク
- **歯車アイコン**: [設定モーダル](#設定モーダル)を開く

## 設定モーダル

ヘッダーの歯車アイコンから開きます。設定は `~/.claude-dashboard/settings.json` に保存されます。

| セクション | 内容 |
|---|---|
| アーカイブ保持期間 | 30 / 90 / 120 / 150 / 180日 / 無制限（既定）。期間より古いアーカイブは次回同期時に削除 |
| セッション取り込みソース | Codex CLI / Gemini CLI の取り込み ON/OFF（Claude Code は常に有効） |
| AI分析プロバイダ | 振り返り・優先課題分析に使うバックエンドの選択と、プロバイダごとの設定（モデル・CLIパス・Base URL・APIキー） |

取り込みソースは「表示・集計の対象」の設定、AI分析プロバイダは「分析の実行エンジン」の
設定で、互いに独立です。プロバイダごとの設定は切り替えても保持されます。

## AI分析（セッション振り返り・優先課題）

### セッション振り返り

分析はハーネスエンジニアリングの実践に基づく2層構成です。

- **定量メトリクス（LLM 不要・JSONL から機械算出）**: 推定変更行数・編集/再編集ファイル数（手戻り）、
  ツールエラー率・テスト実行/失敗回数（不具合）、ユーザー割り込み回数、
  工数効率（行/時間）・コスト効率（$/100行）・キャッシュ読取比率（節約）。
  分析結果に同梱保存され、セッション詳細とダッシュボードの週次トレンドに表示されます。
- **LLM 定性評価**: ハーネス実践5軸スコア（計画・タスク分解 / コンテキスト提供 /
  検証・テスト / 軌道安定性 / スコープ規律、各1〜5）と、手戻り原因ベースの
  カテゴリ付き改善アクション（次のセッションでそのまま実行できる一文）。
  定量メトリクスはプロンプトに注入され、評価の根拠として使われます。

### 優先課題の分析

保存済みの振り返り（直近最大20セッション分）を横断して優先度の高い課題を1〜3件選び、
リポジトリ内蔵のベストプラクティスカタログ（`src/lib/analysis/practices.ts`。
CLAUDE.md 整備・計画モード・TDD などのハーネス実践9件と、完了条件の明文化・
タスク分解などのプロジェクト管理実践4件）を根拠にした構造化アクションを提案します。

各アクションは実施手段の種別（依頼プロンプト / CLAUDE.md / ワークフロー / 設定・ツール）、
根拠プラクティス、実施手順、実数値を引用した期待効果、コピペしてそのまま使える
スニペットを持ちます。カタログは入力の改善点カテゴリ頻度に応じて選択的に
プロンプトへ注入されます。プロジェクト別の優先課題分析にも対応しています。

### 分析プロバイダ

| プロバイダ | 方式 | 備考 |
|---|---|---|
| Claude Code CLI（既定） | `claude -p` ヘッドレス | コスト表示あり。モデルは haiku / sonnet |
| Codex CLI | `codex exec` ヘッドレス | `--output-schema` で構造化出力 |
| Gemini CLI | `gemini -p` ヘッドレス | スキーマはプロンプト埋め込み |
| LM Studio | OpenAI 互換 REST（既定 `http://localhost:1234/v1`） | ローカルモデル。APIキー不要 |
| OpenAI 互換 API | OpenAI 互換 REST | Ollama / vLLM など。Base URL・APIキーを設定 |

**APIキーの注意**: `settings.json`（パーミッション 600）に平文で保存されます。
API レスポンスにはキー本体は含まれません（`hasApiKey` のみ）。平文保存を避けたい場合は
環境変数 `OPENAI_COMPAT_API_KEY` を使ってください。

### 旧形式の分析結果

旧形式（セッション振り返りは schemaVersion 1、優先課題は v1/v2）の分析結果は
「stale / 旧形式」として案内され、再分析で新形式に更新できます。

## アーカイブ（自動バックアップ）

サーバー起動中、5分間隔（`ARCHIVE_SYNC_INTERVAL_MS` で変更可）で全ソースのログを
`~/.claude-dashboard/archive` へ自動ミラーします。

- CLI 側でログが削除されても（Gemini CLI の30日自動削除、Claude Code のクリーンアップ等）、
  アーカイブ済みセッションは引き続き閲覧・集計できます
- 保持期間（設定モーダル）を過ぎたアーカイブは次回同期時に削除されます
- セッションが消滅した場合、対応する分析結果も掃除されます
- ソース別の全滅ガード: あるソースのライブログが一斉に消えた場合、
  誤削除とみなしてそのソースのアーカイブ削除を保留します

## 環境変数

すべて任意です（未設定なら既定値で動作します）。

### データの場所

| 変数 | 既定値 | 説明 |
|---|---|---|
| `CLAUDE_DATA_DIR` | `~/.claude/projects` | Claude Code のログディレクトリ |
| `CODEX_DATA_DIR` | `~/.codex/sessions` | Codex CLI のロールアウト（セッション）ディレクトリ |
| `CODEX_ARCHIVED_DIR` | `~/.codex/archived_sessions` | Codex CLI がアーカイブしたセッションのディレクトリ |
| `GEMINI_DATA_DIR` | `~/.gemini/tmp` | Gemini CLI のチャット記録ディレクトリ（`<hash>/chats/session-*.json(l)`） |
| `CLAUDE_ARCHIVE_DIR` | `~/.claude-dashboard/archive` | アーカイブ（ミラー）の保存先 |
| `CLAUDE_SETTINGS_PATH` | `~/.claude-dashboard/settings.json` | ダッシュボード設定の保存先 |
| `CLAUDE_ANALYSIS_DIR` | `~/.claude-dashboard/analysis` | AI分析結果の保存先 |

### 集計・取り込み

| 変数 | 既定値 | 説明 |
|---|---|---|
| `IDLE_THRESHOLD_MS` | `300000`（5分） | 操作時間推定のアイドル閾値。レコード間隔がこれを超えると離席とみなし不算入 |
| `MAX_FILE_SIZE_MB` | `100` | この上限を超える JSONL はスキップ（警告ログ出力）。メモリ枯渇防止 |
| `ARCHIVE_SYNC_INTERVAL_MS` | `300000`（5分） | アーカイブ自動同期の間隔 |

### AI分析

| 変数 | 既定値 | 説明 |
|---|---|---|
| `CLAUDE_CLI_PATH` | `claude` | Claude Code CLI のパス（設定画面の CLIパス が空のとき使用） |
| `ANALYSIS_TIMEOUT_MS` | `180000`（3分） | AI分析1回のタイムアウト |
| `ANALYSIS_MAX_BUDGET_USD` | `1` | Claude Code CLI での分析1回の予算上限 |
| `OPENAI_COMPAT_API_KEY` | なし | OpenAI互換APIのキー。設定されていれば settings.json の apiKey より優先 |

## 単価改定・新モデル追加

`src/lib/pricing/model-pricing.ts` の `PRICING_TABLE` を編集するだけで完結します
（USD / 1M tokens。input / output / 5m・1h キャッシュ書込 / 読取の5係数）。
単価表にないモデルはモデル名の部分一致（fable/opus/sonnet/haiku）でフォールバックし、
UI に「推定」バッジが表示されます。

## 集計の仕様（重要ポイント）

- **requestId デデュープ**: 同一 API リクエストが content block 毎に複数行記録されるため、
  usage は `requestId`（欠落時は `message.id` → `uuid`）で1回のみ計上。
  これを行わないとコストが約2.3倍に過大計上される（実測検証済み）。
- **ターン分割**: user プロンプト（`promptId`）単位。`tool_result` のみの行や
  `<local-command-caveat>` 等のメタ行は新ターンにしない。
- **所要時間**: Claude Code が記録する `system/turn_duration` を優先、なければタイムスタンプ差分。
- **サブエージェント**（`isSidechain`）: コストには算入、メッセージ数は別枠表示。
- **キャッシュ**: ファイルの mtime+size ベースのインメモリキャッシュ。変更時のみ再パース。
- 週は月曜開始、タイムゾーンはシステム TZ。

## セキュリティ

- サーバーは `127.0.0.1` のみにバインド（認証なしのため）
- 全リクエストで Host ヘッダを検証し、ループバック
  （`127.0.0.1` / `localhost` / `[::1]`）以外を 403 で拒否します
  （DNSリバインディング対策、`src/proxy.ts`）
- `-H` で LAN 公開する場合は `src/lib/api/host.ts` の許可リストにホスト名の追加が必要です
- データは一切外部送信されません（AI分析をリモート API プロバイダに向けた場合を除く）

### コード構成

| パス | 内容 |
|---|---|
| `src/app/` | Next.js のページと API ルート |
| `src/components/` | UI コンポーネント（テーブル・チャート・モーダル） |
| `src/lib/parser/` | JSONL パーサ |
| `src/lib/sources/` | ソース別（claude / codex / gemini）の探索・変換 |
| `src/lib/domain/` | ターン分割・セッション構築 |
| `src/lib/aggregate/` | サマリー・時系列・モデル/ツール別などの集計 |
| `src/lib/pricing/` | モデル単価表とコスト計算 |
| `src/lib/analysis/` | AI分析（メトリクス・プロバイダ・キュー・優先課題） |
| `src/lib/archive/` | アーカイブ同期とスケジューラ |
| `src/lib/view/` | 一覧の絞り込み・並べ替えロジック |
| `src/lib/store/` | ファイル読み込みとキャッシュ |

データレイヤー（`src/lib/`）は Next.js 非依存の純関数で構成され、
`tests/unit/` に1対1対応のテストがあります。fixture（`tests/fixtures/`）は
実データを含まない匿名化した合成 JSONL です。
